ClearCost API

A unified REST API for managing your inventory, materials, recipes, and business data. All endpoints use the same base pattern with store-specific parameters.

Authentication

All requests require a Bearer token in the Authorization header. Obtain a token by signing in through the app; the Supabase access token is used directly.

Authorization: Bearer YOUR_ACCESS_TOKEN
Content-Type: application/json

Unauthenticated requests return 401 Unauthorized.

Base URL

All data endpoints use a single base path:

/api/data

Operations are specified via query parameters: store, action, and optionally id.

Error Handling

Errors return JSON with an error field:

{ "error": "Session expired" }
StatusMeaning
200Success
401Unauthorized (invalid or expired token)
400Bad request (missing params)
500Server error

List Records

GET /api/data?store={store}&action=list

Returns all records for the authenticated user in the specified store.

curl -H "Authorization: Bearer $TOKEN" \
  "https://yourapp.com/api/data?store=products&action=list"

Get Record

GET /api/data?store={store}&action=get&id={id}

Returns a single record by ID.

curl -H "Authorization: Bearer $TOKEN" \
  "https://yourapp.com/api/data?store=products&action=get&id=42"

Create Record

POST /api/data?store={store}&action=create

Creates a new record. Send the record data as JSON body.

curl -X POST -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"Widget A","quantity":50,"sellPrice":12.99}' \
  "https://yourapp.com/api/data?store=products&action=create"

Update Record

POST /api/data?store={store}&action=update&id={id}

Updates an existing record. Send only the fields to update.

curl -X POST -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"quantity":75}' \
  "https://yourapp.com/api/data?store=products&action=update&id=42"

Delete Record

POST /api/data?store={store}&action=delete&id={id}

Permanently deletes a record by ID.

curl -X POST -H "Authorization: Bearer $TOKEN" \
  "https://yourapp.com/api/data?store=products&action=delete&id=42"

Upsert Record

POST /api/data?store={store}&action=upsert

Creates a record if it does not exist, or updates it if it does (matched by ID).

curl -X POST -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"id":42,"name":"Widget A","quantity":100}' \
  "https://yourapp.com/api/data?store=products&action=upsert"

Batch Create

POST /api/data?store={store}&action=batch

Creates multiple records at once. Send a records array in the JSON body.

curl -X POST -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"records":[{"name":"A","quantity":10},{"name":"B","quantity":20}]}' \
  "https://yourapp.com/api/data?store=products&action=batch"

Business Profile

GET /api/data?store=businesses&action=profile

Returns the authenticated user's business profile (name, theme, thresholds, labels).

POST /api/data?store=businesses&action=profile

Updates the business profile. Send only the fields to change.

curl -X POST -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"My Workshop","product_label":"Candle"}' \
  "https://yourapp.com/api/data?store=businesses&action=profile"

Store Reference

Available stores: products materials recipes suppliers orders locations production expenses history waste batches transactions

Products

Finished goods you sell or track inventory for.

FieldTypeDescription
namestringProduct name (required)
quantitynumberCurrent stock count
sellPricenumberSelling price in dollars
skustringSKU or barcode identifier
notestringOptional note
needsMadebooleanFlagged as needing production
inProductionbooleanCurrently being produced
lowThresholdnumberCustom low-stock threshold (overrides global)
locationIdnumberWarehouse/location ID

Materials

Raw materials, packaging, labels, and supplies.

FieldTypeDescription
namestringMaterial name (required)
categorystringraw, packaging, label, fragrance, other
unitstringunits, lbs, oz, kg, ml, each
quantitynumberAmount on hand
costPerUnitnumberCost per unit in dollars
supplierIdnumberLinked supplier ID
lowThresholdnumberLow-stock alert threshold

Recipes

Bill of materials linking products to their required materials.

FieldTypeDescription
namestringRecipe name
productIdnumberOutput product ID
yieldnumberUnits produced per batch
ingredientsarrayArray of {materialId, quantity} objects

Suppliers

Vendor and supplier contact information.

notestringNotes
